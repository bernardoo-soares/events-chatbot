# AGENTS.md

## Docs

Docs explain architecture, plans, implementation history, and delivery notes. Keep them useful for
future maintainers rather than duplicating every code detail.

## Rules

- Update docs when setup, architecture, or major behavior changes.
- Keep sprint logs factual: files changed, behavior implemented, checks run, known issues.
- Do not include secrets, private keys, or raw API payload dumps.
- Prefer concise architecture diagrams and explicit boundaries.
- If docs conflict with code, either update docs or call out the drift.

## Important Files

- `architecture.md`: high-level system design and boundaries.
- `technical_plan.md`: concrete code design.
- `implementation_plan.md`: sprint plan and delivery guardrails.
- `ai_usage.md`: AI-assisted development notes.
- `logs/`: sprint history.
